The scope of VAR , LET , and CONST
In JavaScript, there are three ways of declaring variables, var, let and const. They have different scope and behavior.
Here's a breakdown of their scopes and characteristics:

var:
The "var" statement is used to create variables within a function or globally, and you can choose to give them initial values. These variables can be accessed from anywhere within a function, and you can also declare them again if needed. 
function example() { 
 if (true) {
     var localVar = "I'm inside ";
     console.log(localVar); 
 // Output: I'm inside 
 }
 console.log(localVar); 
// Output: I'm inside   
}  
example();  
//Output:ReferenceError  
console.log(localVar);
Like the code, var can be declared in a block or a function and it is hoisted at the top of the function and it can be accessed from anywhere in the function but not outside the function.

let:
When a variable is declared with let it can be accessed within the scope only where it is declared. Outside the block it will show undefined. And let can be reinitialized but not re-declared. 
 
function example() {
  if (true) {
    let localVar = "I'm inside ";
    console.log(localVar);
    // Output: I'm inside 
  }
  //  not accessible 
    console.log(localVar);
  }


const:
The "const" declaration is used to create variables with block-level scope, meaning they are only accessible within the block of code where they are defined. Once you assign a value to a constant, you cannot change that value by reassigning it with the assignment operator. However, if a constant is an object, you can still modify its properties by adding, updating, or removing them. 
function example() {
  const pi = 3.14159; 
  // Declare a constant variable
  if (true) {
   const localVar="I'm inside";
    console.log(localVar); 
  // Output: I'm inside
  }
  console.log(pi); 
  // Output: 3.14159 
 // not accessible
 console.log(localVar);
}

Use cases of NULL and UNDEFINED
Null:
In JavaScript, "null" is considered an object, and it can be assigned to a variable to represent the absence of a value. JavaScript doesn't automatically assign a value of null; you need to do it in your code. For instance, when you don't have an initial value for a variable, you can assign null to it. Let's look at an example. 
// usage of null in JavaScript
var n = null;
// later in your program
n = array.slice(0, 2);
// example
Here, in the example above, the variable n is given an initial value of null. Later, in the program, the value of the variable has been calculated from an expression.

Undefined:
"Undefined" in JavaScript indicates that a variable has been declared but has not yet been given a value. It's a distinct data type in JavaScript, known as "undefined." When you declare a variable but don't assign it a value, JavaScript automatically initializes it with the default value of "undefined". 
// usage of undefined in JavaScript
var n ;
console.log(n);
//undefined
Here as the variable is declared but not assigned to any value, the variable by default is assigned a value of undefined.
REST API

REST API is a way of accessing web services in a simple and flexible way without having any processing.
REST technology is generally preferred to the more robust Simple Object Access Protocol (SOAP) technology because REST uses less bandwidth, simple and flexible making it more suitable for internet usage. After that, a response comes back from the server in the form of a resource which can be anything like HTML, XML, Image, or JSON. But now JSON is the most popular format being used in Web Services.
